# Security Policy

KawanuaDev takes the security of our software seriously, including all source code repositories managed through our GitHub organisation. If you believe you have found a security vulnerability in this repository, please report it to us as described below.

## Reporting Security Issues

Please do not report security vulnerabilities through public GitHub issues. Instead, please use GitHubs private vulnerability reporting functionality associated to this repository. Additionally, you may email us with all security-related inquiries and notifications at `security@kid.or.id`.

Please include the requested information listed below (as much as you can provide) to help us better understand the nature and scope of the possible issue:

1. Type of issue
2. Step-by-step instructions to reproduce the issue
3. Proof-of-concept or exploit code (if possible)
4. Potential impact of the issue, including how an attacker might exploit the issue

This information will help us triage your report more quickly. Any and all CVEs will be requested and issued through GitHubs private vulnerability reporting functionality, which will be published alongside the disclosure.

This security policy only applies to the most recent stable branch of App Kawanua. Flaws in old versions that are not present in the current stable branch will not be fixed.

## Supported Versions

Only the most recent stable branch of App Kawanua is eligible for security fixes. Flaws in older versions that are not present in the current stable branch will not be fixed.

## Disclosure Policy

We will disclose security vulnerabilities in accordance with the following timeline:

1. When a security vulnerability is reported, we will acknowledge receipt of the report within 24 hours.
2. We will provide an estimated timeline for resolution within 5 business days.
3. We will notify you when the vulnerability has been fixed and is ready for public disclosure.
4. We will coordinate with you to determine the appropriate time for public disclosure.

## Third-Party Dependencies

We use a variety of third-party dependencies in our project. We will update these dependencies on a regular basis to ensure that we are using the latest versions. We will also monitor for any security vulnerabilities in our dependencies and will update them as soon as possible.

## Contact

If you have any questions about our security policy, please contact us at `security@kid.or.id`.
